Documentation
Overview
BoardJS is a JavaScript library that simplifies the way of making interactive presentations, stories and ads.
The library works by writing commands such as adding texts, images, moving objects and showing tooltips, and once the .go() function is called, commands will be executed one by one. The list of commands will be interpreted in the library as a timeline.

The benefits and features of the library are:
  • 1. Optimized and fast
  • 2. Cross browser and platform
  • 3. Changes the presentation without refreshing the page
  • 4. Applies commands to more than one object at a time
  • 5. Easy to update presentations
  • 6. Uses only HTML, JavaScript and CSS
Take a look at the examples.
If you get familiar with commands, any presentation should take about 30 minutes to finish the code.
What will take your time is thinking about the presentation and preparing its content.

Preparation
Preparing the library is quite simple, creating a new instance by only one line of code with one parameter, like this:
var myObject = new BoardJS("ContainerDiv");

ContainerDiv is the id of the handle div (the container which will handle the objects)
Now you can start writing commands.
Note: you can clean the board by using myObject.reset(), so you don't have to prepare the board again.

Using Commands
After you prepare the board, you can write your commands then execute them, like this:
var myObject = new BoardJS("ContainerDiv");
myObject.background("rgb(78,122,219)", 500);
myObject.addText("Hello World", 0, 50, "title center", "fade", 1000, "linear");
myObject.go();
Run the code


Commands
addText
  • Syntax:
    .addText(TheText, [x],[y], [ClassName], [ShowType], [Duration], [Easing], [ID], [Freeze])
  • TheText: the content of the object.
  • x: left position, the default is 0.
  • y: top position, the default is 0.
  • ClassName: className of object, you can use this to add additional styles.
  • ShowType: the way you want to show the object, ShowTypes are listed below.
  • Duration: the time it takes to show the object.
  • Easing: the type of the animation's formula. For a clear understanding, see the Easing example.
  • ID: the id of the object, you will need it to control the object in other commands.
    caution: be careful with ID, it could conflict with other ids in the page.
  • Freeze: can be true or (false or null); use this if you want to freeze the timeline until the object is fully shown.
addImage
  • Syntax:
    .addImage(ImageSRC, [x], [y], [w], [h], [ShowType], [Duration], [Easing], [ID], [Freeze])
  • ImageSRC: the source of the image.
  • x: left position, the default is 0.
  • y: top position, the default is 0.
  • w: width; if the value is null or empty string, the width will be auto.
  • h: height; if the value is null or empty string, the height will be auto.
  • ShowType: the way you want to show the object, ShowTypes are listed below.
  • Duration: the time it takes to show the object.
  • Easing: the type of the animation's formula.
  • ID: the id of the object, you will need it to control the object in other commands.
  • Freeze: if true, the timeline will remain frozen until the object is fully shown.
delay
  • Syntax:
    .delay(HowLong)
  • Freezes the timeline for a specified period of time.
  • HowLong is a numeric value in milliseconds (1000 = 1 second)
background
  • Syntax:
    .background (ImageSRC | HexColor | RGBColor | RGBAColor, [VisibilityDuration], [Freeze])
  • Changes the background of the board, you can set a color or an image background.
  • ImageSRC: source of the image.
  • HexColor: to set a color with a hex value (ex: #0099ff)
  • RGBColor: to set a color with an RGB value (ex: rgb(0, 128, 255))
  • RGBAColor: to set a color with an RGBA value (ex: rgba(0, 128, 255, 0.5))
  • VisibilityDuration: the time it takes to show the background using fade effect with a linear movement.
  • Freeze: if true, the timeline will remain frozen until the background is fully shown.
flash
  • Syntax:
    .flash(IDs, HowMany, [Duration], [Freeze])
  • Following on from this command, all of the commands are for controlling the objects and the timeline.
  • This command is for flashing the selected object(s), it fades it in and out for several times depending on the "HowMany" value.
  • IDs: id of the target, IDs means that you can specify more than one target (ex: "id1 id2 id3")
  • HowMany: specifies the times you want to apply the flash. If the value is "always", it will flash forever.
  • Duration: the duration of each fading process in milliseconds.
  • Freeze: if true, the timeline will remain frozen until the flash is finished.
fadeOut
  • Syntax:
    .fadeOut(IDs, Duration, [Freeze])
  • Hide the target by fading it smoothly.
  • IDs: the target.
  • Duration: the duration of the fading process in milliseconds
  • Freeze: if true, the timeline will remain frozen until the target is hidden.
fadeIn
  • Syntax:
    .fadeIn(IDs, Duration, [Freeze])
  • Shows the target by fading it smoothly.
  • IDs: the target.
  • Duration: the duration of the fading process in milliseconds
  • Freeze: if true, the timeline will remain frozen until the target is fully shown.
moveTo
  • Syntax:
    .moveTo(IDs, [x], [y], [Duration], [Easing], [Freeze])
  • Moves the target from its position to a new one.
  • IDs: the Target.
  • x: the new left position. If the value is null or empty, the left position will remain the same.
  • y: the new top position. If the value is null or empty, the top position will remain the same.
  • Duration: the time it takes to finish the movement.
  • Easing: the type of the animation's formula.
  • Freeze: freezes the timeline until this command is completed.
remove
  • Syntax:
    .remove(IDs)
  • Removes the target.
  • IDs: the target
addTooltip
  • Syntax:
    .addTooltip(IDs, Content)
  • Adds a tooltip to the target to be shown when you move the cursor upon it.
  • IDs: the target
  • Content: the content of the tooltip.
removeTooltip
  • Syntax:
    .removeTooltip(IDs)
  • Removes the recently added tooltip.
  • IDs: the target
showTooltip
  • Syntax:
    .showTooltip(IDs, Content, [LifeTime], [Freeze])
  • Shows a tooltip above the target for a specified period of time.
  • IDs: the target
  • Content: the content of the tooltip.
  • LifeTime: specifies the time you want the tooltip to be visible. If this value is "always", it will still visible until you use removeTooltip.
  • Freeze: freezes the timeline until the tooltip is removed.
html
  • Syntax:
    .html(IDs, Content)
  • Changes the content of the text object.
  • IDs: the target
  • Content: the content of the object.
setStyle
  • Syntax:
    .setStyle(IDs, Style)
  • Changes the style of the object.
  • IDs: the target
  • Style: the new style, the syntax is the same of the inline style (ex: "border:2px solid #000;margin:10px")
changeImage
  • Syntax:
    .changeImage(IDs, ImageSRC)
  • Changes the image of the image object.
  • IDs: the target
  • ImageSRC: the source of the image.
addEvent
  • Syntax:
    .addEvent(IDs, TheEvent, TheFunction)
  • Registers functions with any event of the target such as click.
  • IDs: the target
  • TheEvent: click, mouseover...; note: don't use the prefix "on".
  • TheFunction: the function you want to call, such as: function(){alert("Hello World")}
removeEvent
  • Syntax:
    .removeEvent(IDs, TheEvent)
  • Removes the recently registered event of the target.
  • IDs: the target
  • TheEvent: the event you want to remove.
hover
  • Syntax:
    .hover(IDs, Style, [Duration], [Easing])
  • Changes the style of the target on hovering, same as CSS :hover.
  • IDs: the target.
  • Style: the style to be applied when hovering with the addition of "scale" and "rotate", such as "rotate:100;scale:2;border:2px solid"
  • Duration: the time it takes to fully apply the new styles.
  • Easing: the type of the animation's formula.
unHover
  • Syntax:
    .unHover(IDs)
  • Cancels the hover command of the target.
  • IDs: the target.
merge
  • Syntax:
    .merge(IDs, MergeWith)
  • Merges the target with another object and you will still can control it.
  • IDs: the target.
  • MergeWith: the object you want the target to be merged with.
unMerge
  • Syntax:
    .unMerge(IDs)
  • Unmerges the recently merged objects.
  • IDs: the target
rotate
  • Syntax:
    .rotate(IDs, Angle, [Duration], [Easing], [Freeze])
  • Rotates the target with a specified angle with ease, requires css3.
  • IDs: the target
  • Angle: the angle in degrees.
  • Duration: the time it takes to fully apply the rotation.
  • Easing: the type of the animation's formula.
  • Freeze: freezes the timeline until the rotation is finished.
zoom
  • Syntax:
    .zoom(IDs, Scale, [Duration], [Easing], [Freeze])
  • Scales the target with a specified scale value with ease, requires css3.
  • IDs: the target
  • Scale: the Scale value, 1 is the original size.
  • Duration: the time it takes to fully apply the scaling.
  • Easing: the type of the animation's formula.
  • Freeze: freezes the timeline until the scaling is finished.
circle
  • Syntax:
    .circle(IDs, [AdditionalStyles])
  • Applies several styles to the target to be like a circle, requires css3.
  • IDs: the target.
  • AdditionalStyles: extra parameter, same as setStyle command
round
  • Syntax:
    .round(IDs, TheValue)
  • Sets the borderRadius of the object, requires css3.
  • IDs: the target.
  • TheValue: the value of the rounded border corners.
playSound
  • Syntax:
    .playSound(SoundSRC,[Freeze])
  • Plays a sound file using the powerful JavaScript library SoundManager2
  • SoundSRC: the source of the sound file.
  • Freeze: freezes the timeline until the sound is finished.
stopSound
  • Syntax:
    .stopSound([SoundSRC])
  • Stops the sound.
  • SoundSRC: the source of the sound file. If this value is null or an empty string, all playing sounds will be stopped
go
  • Syntax:
    .go()
  • Prepares the written commands and begins the show.
  • The presentation will not start until you use this command.
goTo
  • Syntax:
    .goTo(CommandID)
  • Written commands have IDs depending on their sequence.
  • This command orders the board to jump to a specified command and and continue normally.
  • Example:
myObject.addText("Text 1") // This command's id is 0
myObject.addText("Text 2") // This command's id is 1
myObject.addText("Text 3") // This command's id is 2
myObject.goTo(1) // The board will jump to command 1


reset
  • Syntax:
    .reset()
  • Stops and cleans the board.
  • Always use this command at the first line of the presentation's code.
pause
  • Syntax:
    .pause()
  • Pauses the timeline, any working commands will remain working until they finish
resume
  • Syntax:
    .resume()
  • Continues the timeline. Works only if the board is paused.
Easings
  • Available easings are:
  • linear - easeIn - easeOut - backIn - backOut - elastic - bounce
ShowTypes
  • Available ShowTypes are:
  • Appear - Split - Fade - Zoom - FromLeft - FromRight - FromTop - FromBottom
  • Split works only with text objects.
  • You can apply more than one ShowType to an object like: "fade fromBottom"
Sound issue
Nothing in this world is free from issues.
I am always having problems with sounds, even if I am using the cross browser and platform library soundManager2
In iOS, playing sounds by code without user interference such as onClick is forbidden.
Click here for details (section: User Control of Downloads Over Cellular Networks)

But the sound works normally with events such as onClick.

Note: If the presentation has no sounds, the board does not ask for soundManager2.
So, if you don't have sounds in your presentations, you don't have to use soundManager2.js at all.

Conclusion
The main purpose of BoardJS is to reduce the time of making interactive presentations and to make sure they will work in most browsers.
The Examples use most of the commands, you can find the source in main.js.
Examples are tested on:

Windows:
  • Chromium 21
  • Google Chrome 7
  • Internet Explorer 7 / 8 / 9
  • Firefox 3.5.5 / 3.6.6 / 4 / 17
  • Safari 4.0.3 / 5.1.7
  • Opera 11 / 12.11
Mac OS:
  • Safari 5.1.6
  • Google Chrome 23
  • Firefox 6.0.1
Touch devices:
  • Android 4.0.3 (Samsung Galaxy S2)
  • Android 2.3.6 (Samsung Galaxy Ace 2)
  • iPod 4th generation with iOS 6.0 (Sound works with user interaction only)
  • iPad 3rd generation with iOS 6.0.1 (Sound works with user interaction only)

And of course, Internet Explorer 7 and 8 do not support css3, but the show continues without any interruption.
